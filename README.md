# README №2: Технічна документація для розробників (Dev Doc)

Цей документ описує архітектуру проєкту, призначення модулів та структуру внутрішніх даних.
## ⚙️ Технологічний стек
* **Мова**: Python 3.10+
* **Бібліотеки**: `requests`, `pandas`, `xlsxwriter`, `python-dotenv`.
* **Формат обміну**: JSON (Allegro API v1).

## 📂 Опис модулів (файлів)

* **main.py**: Точка входу. Координує роботу всіх модулів: перевіряє токени, запускає парсинг, передає дані в мапер та ініціює експорт.
* **config.py**: Завантажує ключі з `.env`.
* **auth.py**: Модуль авторизації. Містить логіку отримання `access_token` через `authorization_code` та механізм автоматичного оновлення через `refresh_token`. Зберігає дані в `token.json`.
* **core_mapper.py**: "Серце" парсингу. Витягує з сирого JSON від Allegro тільки потрібні поля. Формує проміжний файл `core.json`.
* **horoshop_exporter.py**: Трансформатор. Читає `core_base.json`, витягує EAN з параметрів, визначає пріоритет Артикула (EAN vs Signature), обробляє блокову структуру опису (description) та зшиває її в HTML, після чого генерує `xlsx` файл за шаблоном Хорошопу.

# 📌 Структура CORE-файлу (core.json)
Цей файл генерується мапером (`core_mapper.py`) і містить "причесані" дані з Allegro API. Це єдине джерело правди для подальшого експорту в Хорошоп або інші CRM.

## Основні ідентифікатори
* **`allegro_signature`**: Внутрішній артикул (SKU) складу. Головний ключ для синхронізації цін/залишків.
* **`allegro_id`**: 11-значний унікальний ID оффера на самому Allegro.

## Базові дані товару
* **`name`**: Назва товару.
* **`remainder`**: Доступний залишок на складі.
* **`price`** / **`currency`**: Поточна ціна та валюта (напр., PLN).
* **`images`**: Масив посилань на фотографії (перше фото — головне).
* **`description`**: Масив секцій опису. Потребує подальшого парсингу в суцільний HTML для Хорошопа.

## Характеристики та Категорії
* **`parameters`**: Повний масив характеристик. **Важливо:** саме тут лежить штрихкод (`EAN`) та Бренд.
* **`allegro_category`**: ID категорії на Allegro (для мапінгу з нашими розділами).
* **`size`**: ID таблиці розмірів (якщо є).

## Статуси та Продаж
* **`stock_status`**: Статус публікації (`ACTIVE`, `INACTIVE` тощо). Використовується для фільтрації знятих з продажу товарів.
* **`auctions`**: Формат продажу (відсікаємо аукціони, залишаємо `BUY_NOW`).
* **`b2b`**: Ознака товару "тільки для бізнесу" (опт).
* **`price_for_other_countries`**: Ціни для додаткових ринків (Чехія, Словаччина).

## Метадані та Логістика
* **`VAT`**: Ставки податків (ПДВ).
* **`location`**: Місцезнаходження товару (місто, країна).
* **`createdAt`** / **`updatedAt`**: Дати створення та останнього оновлення.
* **`contact`**, **`attachments`**, **`language`**, **`errors`**, **`discounts`**: Додаткова системна інформація.
